v0.28 Benchmark Harness Expansion

Evaluation now tracks product effectiveness across attack, benign, compatibility, and evidence cases.

Goal

Move beyond isolated unit checks by registering product-level suites, cases, metrics, and optional heavy validation boundaries.

Suite Categories

CategoryPurpose
AttackSecret egress, unsafe deletion, CI/deploy mutation, and external instruction hijack.
BenignLow-risk commands and repo inspection should keep autonomy.
CompatibilityAgent harness artifacts should remain compatible under Kappaski control.
EvidenceBundles, proof, replay, coverage, and audit reports must verify.

Metrics

block_rate, approval_rate, benign_false_positive_proxy, latency_overhead_ms, proof_completeness, and coverage_distribution.

CLI

kappaski eval list
kappaski eval benchmark --suite v0.28-harness-expansion
kappaski eval report --suite full-product-readiness --out .kappaski/full-product.json

Tests

PYTHONPATH=src /usr/bin/python3 -m pytest -q tests/test_core.py::test_v028_benchmark_harness_registry_expands_product_metrics
PYTHONPATH=src python3 -m kappaski.cli eval benchmark --suite v0.28-harness-expansion

Boundary

Default validation is deterministic and local. Full public SWE-Bench Lite execution and other external suites remain optional heavy validation and skip cleanly when dependencies are missing.
