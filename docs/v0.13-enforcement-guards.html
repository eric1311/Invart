v0.13 Enforcement Guards

v0.13 establishes the first Rust-native enforcement direction while keeping the Python control plane as the policy and audit authority. The local slice includes deterministic guard checks plus a Rust file-write shim source contract, build command, direct shim decisions, wrapper-level execution interception, adapter/Claude bridge opt-in enforcement, and deterministic fallback when a native shim binary is unavailable or platform-incompatible.

Implemented Scope

  • Enforcement order is file-write guard, then env/secrets guard, then network egress guard.
  • Default enforcement failure mode is fail-open with critical alert.
  • Local guard checks classify destructive file operations, secret exposure, and risky network egress before stronger native interception exists.
  • rust/kappaski-shim contains the first Rust file-write shim source. It reads an event/command payload and emits a JSON policy decision.
  • kappaski enforce shim-decision runs the compiled shim against an event before execution.
  • If the compiled shim is missing, cannot build, or cannot execute in the current container/host platform, Kappaski falls back to the deterministic Python file-write guard and marks the decision with fallback: true.
  • kappaski enforce run-file-write is the first wrapper-level interception path: deny blocks before execution, allow executes, and ledger/outcome evidence is recorded when a ledger is supplied.
  • kappaski adapter run --enforcement file-write and kappaski adapter claude-code --enforcement file-write route child command execution through the same shim path.
  • kappaski enforce rust-build-check --skip-if-unavailable validates the Rust crate when cargo is installed and skips cleanly in Python-only environments.

Architecture Slice

Python Control Plane

Policy evaluation, deterministic fallback, ledger/proof integration, benchmark checks, and CLI reporting remain here.

Rust Shim

Small native boundary for file-write decision requests. The source compiles in this workspace with Rust 1.95.0 and is wired into the wrapper-level run-file-write path.

Future Hardening

Kernel/OS-level file/network/process interception remains an explicit product boundary; wrapper enforcement and process-group supervision are implemented.

In the current workspace cargo is available and kappaski enforce rust-build-check passes with cargo check. Containerized tests also verify that a host-built incompatible shim binary does not silently allow unsafe commands; it falls back to deterministic guard evidence.

Boundary

This version is still a local CLI/control-plane slice. It proves the contract, tests, native source, Rust compilation path, deterministic fallback, and wrapper-level file-write interception, but it does not yet provide kernel/OS-level file/network/process interception.
