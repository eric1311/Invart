Kappaski v0.8-v0.40 Readiness And Claim Boundaries

This document records local implementation readiness and the boundary between real local product capability, local experiment substrate, and external-validation contracts. v0.30-v0.39 are deterministic experiment infrastructure; v0.40 adds a full SWE-Bench validation contract; neither is evidence that official external benchmark suites have already completed.

Verification Gate

PYTHONPATH=src python3 -m pytest -q
PYTHONPATH=src python3 -m kappaski.cli eval benchmark --suite full-product-readiness
PYTHONPATH=src python3 -m kappaski.cli roadmap status --require-full
PYTHONPATH=src python3 -m kappaski.cli roadmap status --require-external-validation
PYTHONPATH=src python3 -m kappaski.cli release-candidate verify --out-dir .kappaski/rc
scripts/container-test.sh local

--require-full checks local implementation completeness. --require-external-validation intentionally fails until optional upstream/live benchmark validation has been run and recorded.

Pre-release Real-World Demo

The pre-release hardening pass adds a public-source risk mapping demo that links real-world agent/skill risk reports to Kappaski's before/during/after control-plane artifacts. It is intentionally scoped as public-source mapping plus local reproducible demo evidence, not a claim that private incidents were replayed.

kappaski demo real-world-risk-cases --out-dir .kappaski/real-world-risk-demo
kappaski eval benchmark --suite real-world-agent-risk-demo

Main outputs: real-world-risk-sources.json, real-world-risk-demo.html, live adapter audit/replay, pre-v1 path graph, coverage report, proof, ledger, and audit report.

Full-Product Benchmark

The full-product-readiness benchmark runs a cross-version smoke that touches the reviewer corpus, managed harness approval, process supervision, profile review, TeamRun timeline, multi-domain enforcement, audit signoff, native conformance, bridge conformance, MCP stdio broker, coverage HTML export, identity binding, path graph, path-aware policy, unified mediation, profile governance, the pre-v1 demo package, adapter runtime package, policy-as-code, evidence bundle, benchmark registry, RC gate, the v0.30-v0.39 experiment suites, and the v0.40 SWE-Bench full-validation contract.

kappaski eval benchmark --suite full-product-readiness

Claim Integrity

Passing full-product-readiness means local Kappaski product surfaces and local experiment substrates are runnable. It does not mean optional official external benchmark runs have passed. The roadmap and RC reports now expose claim_scope, evidence_level, and external_validation so the distinction is machine-readable.

kappaski roadmap status --require-full
kappaski roadmap status --require-external-validation

Containerized Local Verification

scripts/container-test.sh local runs the same local readiness suite inside Docker: pytest, the v0.40 SWE-Bench contract benchmark, full-product readiness, roadmap full status, and the RC gate. It also confirms that external validation still fails until the full upstream evidence bundle is actually attached.

scripts/container-test.sh local

For the optional heavy SWE-Bench path, use scripts/container-test.sh swe-bench-full gold kappaski_full_20260601 .kappaski/swe-bench-full. That command is not a completed result until the official all-instance run finishes and produces the full report bundle.
