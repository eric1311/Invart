Benchmark Research And Experiment Code Route

How the external benchmark research map becomes v0.30-v0.39 code: corpus adapters, experiment cases, runners, metrics, reports, and paper-ready evidence.

Status: v0.30-v0.39 are implemented as local deterministic experiment infrastructure. v0.40 adds the full SWE-Bench validation contract. Official upstream/live benchmark validation is a separate gate and must not be inferred from local experiment pass results or contract tests.

Starting Baseline

v0.29 provided the benchmark registry, release-candidate gate, evidence bundle, policy-as-code, adapter runtime packages, proof, replay, audit, coverage, and path graph artifacts that v0.30-v0.39 now exercise as experiment infrastructure.

Remaining Layer

Kappaski still needs exact upstream corpus imports and optional heavy runners for official external validation. The current local substrate normalizes benchmark-shaped cases and scores Kappaski-specific outcomes.

Design Rule

External benchmarks provide seeds and credibility. The Kappaski ledger remains the fact source, and reports must separate agent task success from control-plane success.

v0.40 SWE-Bench Rule

Full SWE-Bench evidence must come from external-validation swe-bench-full: complete SWE-bench/SWE-bench test split, all instances required, official report JSON, and per-instance results. The equivalent princeton-nlp/SWE-bench id is accepted. Lite or subset runs are smoke checks only.

Non-Negotiable Implementation Rules

  • External corpus adapters never become the fact source; the ledger remains authoritative.
  • Every imported case must declare source, trust, capability, resource, sink, expected effect, and required evidence.
  • Attack cases should have paired benign controls whenever possible.
  • Optional heavy external runs must skip cleanly in CI and never masquerade as passed.
  • Reports must separate model/agent task success from Kappaski control-plane success.
  • Coverage claims cannot be promoted from observed to mediated or enforced without a real pre-side-effect boundary.

PYTHONPATH=src python -m kappaski.cli experiment paper-suite --out-dir .kappaski/paper-suite
