v0.15 complete

Native Integration Inventory

Machine-readable inventory for agent-native hooks, plugins, extensions, rules, MCP, sandbox, command, and config surfaces.

What Shipped

Scanner

src/kappaski/native.py discovers repo-local native surfaces for Claude Code, Codex, Gemini CLI, Cursor, OpenCode, OpenClaw, and Hermes.

Global Config Control

Global user config scanning is off by default and only runs with --include-global-config.

Installer

native install supports preview/confirm semantics for Claude Code, Codex, and OpenCode, with backup on existing files.

CLI

kappaski native inventory --target /repo
kappaski native inventory --target /repo --include-global-config
kappaski native install --target /repo --agent claude-code
kappaski native install --target /repo --agent claude-code --confirm

Inventory Model

FieldMeaning
agentAgent family such as claude-code, codex, gemini-cli, cursor, or opencode.
discovery_modefull for first-class targets; discovery_only for OpenClaw and Hermes until APIs stabilize.
surfacesPer-surface records for hooks, plugins, extensions, rules, MCP, sandbox, skills, and config.
gradedeclared when a surface is found in config, otherwise none.

Boundary

This version does not query vendor APIs or plugin registries. It is a deterministic local inventory layer that turns native integration assumptions into auditable facts.
