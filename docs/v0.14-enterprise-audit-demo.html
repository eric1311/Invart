v0.14 Enterprise Audit Demo

v0.14 packages Kappaski as an enterprise security-team demo: a repeatable runtime trace for secret leak and unsafe deletion workflows, with ledger-derived proof, replay, and audit reports.

Demo Flow

Before Runtime

The demo starts a managed session and records the target, goal, agent identity, and ledger path as the fact source.

During Runtime

It records sensitive file read, secret-shaped content, external network upload, and destructive deletion command events through the same runtime policy path used by normal Kappaski sessions.

After Runtime

It exports proof JSON, replay HTML, audit JSON, and an enterprise audit HTML report with folded raw evidence.

Artifacts

  • ledger.jsonl: append-only fact source.
  • proof.json: portable summary derived from the ledger.
  • replay.html: detailed runtime replay.
  • audit-report.json: machine-readable enterprise audit summary.
  • audit-report.html: security-team-facing report with metrics, findings, controls, and folded raw proof evidence.

Verification

kappaski demo enterprise-audit --out-dir /tmp/kappaski-demo
kappaski demo enterprise-audit --mode live-adapter --out-dir /tmp/kappaski-demo-live
kappaski eval benchmark --suite v0.14-enterprise-audit-demo

Boundary

The demo has two modes: scripted ledger-derived mode for deterministic examples, and live-adapter mode that ingests Claude-style hook events and blocks an unsafe deletion command through Rust file-write enforcement before generating ledger, proof, replay, audit JSON, and audit HTML. Ledger-backed security signoff is implemented; hosted organization-specific report templates remain future service work.
