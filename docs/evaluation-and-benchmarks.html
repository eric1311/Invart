Validation Philosophy

External benchmarks provide scenario credibility. Kappaski benchmarks measure the control-plane loop they do not cover: decision, approval grade, outcome recording, proof completeness, ledger verification, approval resolution, replay quality, and redaction safety.

v0.9 Harness Compatibility Target

The future full SWE-Bench path should validate Kappaski as a transparent control plane around an existing agent harness, not as a replacement harness. The benchmark should compare a baseline harness run with a Kappaski-wrapped run and require equivalent task orchestration, stable exit behavior, preserved artifacts, and no unexpected command/environment mutation.

DimensionRequirement
CompatibilityThe agent harness can clone, run, test, grade, and emit its normal artifacts under Kappaski.
SafetyKappaski records command, file, network, environment, skill/tool/API, taint, approval, and blocked critical events without letting deterministic critical rules be downgraded.
AutonomyLow-risk steps continue without human interruption; high-risk steps require approval or block according to mode.
ProofLedger, proof, gate, and replay are sufficient to explain what happened after the harness finishes.

Default vs Optional Heavy Tests

Default

Fast pytest and built-in benchmarks use pinned metadata and local artifacts. They must not require Docker, network, or full benchmark runners.

Optional

Future smoke suites may invoke real framework runners such as SWE-Bench harnesses when Docker/data dependencies are available. Missing dependencies should report skipped, not fail the core suite.

v0.40 Full SWE-Bench Contract

Full SWE-Bench evidence now has a dedicated command and stricter acceptance rule. external-validation swe-bench-full defaults to SWE-bench/SWE-bench, split=test, all instances required, and 2294 expected instances. The equivalent princeton-nlp/SWE-bench id is accepted. It validates results/<run_id>.json and results/<run_id>/instance_results.jsonl. The local v0.40-swe-bench-full-validation-contract benchmark tests Kappaski's parser and subset rejection; it is not a completed upstream run.

kappaski external-validation swe-bench-full \
  --python /path/to/swebench-python \
  --predictions-path /path/to/predictions.jsonl \
  --run-id kappaski_full_2026_05_31 \
  --work-dir .kappaski/swe-bench-full \
  --out .kappaski/swe-bench-full/full-validation.json

v0.24 Pre-v1 Control-Plane Benchmark

The pre-v1 suite is the local 1.0 readiness gate. It validates attack coverage, benign autonomy, artifact compatibility, and evidence completeness with deterministic fixtures.

MetricMeaning
block_rateShare of high-risk paths blocked or denied.
approval_rateShare of paths requiring human mediation.
benign_false_positive_proxyLocal proxy for over-blocking benign workflows.
proof_completenessWhether proof includes accountability, graph, coverage, decisions, approvals, and outcomes.
audit_reconstruction_successWhether post-runtime artifacts can reconstruct the run.
