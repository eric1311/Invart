v0.12 TeamRun And Handoff

Multi-user TeamRun records, declared agent identity validation, and handoff taint inheritance.

Implemented Scope

  • TeamRun targets multi-user, multi-agent environments and can be recorded as first-class ledger facts.
  • Agent identity prefers user declaration, validates against adapter/session facts, and can be persisted in the ledger.
  • Default handoff taint inheritance is resource-reference.
  • Session-wide taint inheritance is available through policy profiles for stricter enterprise controls.
  • Blackboard entries, Handoff records, and restrict-only grant delegations are ledger-backed.
  • TeamRun proof export summarizes TeamRun, identity, blackboard, handoff, and grant facts.

Verification

kappaski teamrun create --name demo --user alice --user bob
kappaski teamrun identity --agent claude --declared-by alice --adapter-agent claude
kappaski teamrun handoff --ledger ledger.jsonl --source-agent a --target-agent b --resource tainted:.env
kappaski teamrun blackboard --ledger ledger.jsonl --teamrun demo --author alice --content "Shared finding"
kappaski teamrun delegate-grant --ledger ledger.jsonl --source-agent a --target-agent b --parent-scope repo:read,repo:write --delegate-scope repo:read
kappaski teamrun proof --ledger ledger.jsonl --out teamrun-proof.json
kappaski eval benchmark --suite v0.12-teamrun-handoff

Boundary

This version is an implemented local control-plane product surface. It records the product contract and validates behavior through deterministic tests and built-in benchmarks before deeper native or external harness integration.
