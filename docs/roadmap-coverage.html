Roadmap Coverage Matrix

This page records the current local control-plane status through v0.40. A capability is implemented when it has runnable APIs/CLI, deterministic tests, benchmark coverage, documentation, and explicit product boundaries. External/live benchmark validation is tracked separately from local implementation readiness.

Verification Commands

kappaski roadmap status
kappaski roadmap status --require-full
	kappaski roadmap status --require-external-validation
	kappaski eval benchmark --suite full-product-readiness
python3 -m pytest -q

Status Semantics

StatusMeaning
implementedThe current local product surface is usable, tested, benchmarked, documented, and honest about boundaries.
local experiment substrateThe runner, metrics, proof, and artifacts are implemented locally, but default cases may be simulated or benchmark-shaped rather than official upstream benchmark executions.
product boundaryA known external or future-service limit, such as kernel/OS enforcement, hosted identity roles, or optional heavy benchmark infrastructure. It is not hidden as a missing implementation.
