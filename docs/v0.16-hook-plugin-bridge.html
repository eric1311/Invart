Hook And Plugin Event Bridge

Native hook/plugin payloads are normalized into Kappaski Invocation records and rendered back as vendor-specific allow/block responses.

What Shipped

Normalizer

src/kappaski/native_bridge.py maps Claude Code, Codex, OpenCode, and generic hook payloads into the canonical Invocation model.

Blocking Response

Bridge responses can return deny/block semantics for vendors that support native hook blocking.

Coverage Metadata

Native bridge events carry coverage_layer=native_hook so proof and replay can distinguish hook-level control from stronger shim enforcement.

CLI

kappaski bridge native --agent codex --event '{"tool":"shell","arguments":{"command":"rm -rf ."},"session_id":"codex-session"}'
kappaski bridge native --agent claude-code --event '{"tool_name":"Bash","tool_input":{"command":"echo ok"},"session_id":"claude-session"}'

Decision Semantics

InputOutput
Vendor hook payloadKappaski Invocation with adapter native_hook:<agent>.
High or critical deterministic findingNative response rendered as deny/block where supported.
Low-risk eventNative response rendered as allow.
