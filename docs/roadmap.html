v0.30-v0.39

Benchmark Experiment Code Route

Implemented Local Substrate

Benchmark research is executable experiment infrastructure: ExperimentCase runner, AgentDojo/AgentDyn-shaped adapters, AgentSecBench-shaped authority/data-flow cases, SWE-bench-like friction track, SKILL-INJECT-shaped supply-chain track, secure coding gate, coverage truthfulness matrix, LLM reviewer selectivity, audit tamper assurance, and one paper-ready E0-E6 local result bundle. Official upstream/live benchmark validation is tracked separately and is not claimed as complete.

v0.40

Full SWE-Bench Validation Contract

Implemented Contract

Full SWE-Bench evidence now has a strict CLI contract: external-validation swe-bench-full defaults to SWE-bench/SWE-bench, split=test, all instances required, 2294 expected instances, and validates results/<run_id>.json plus results/<run_id>/instance_results.jsonl. The equivalent princeton-nlp/SWE-bench id is accepted. The local benchmark proves the contract and rejects subset evidence; the actual upstream full run remains a separate external-validation artifact.
