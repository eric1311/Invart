Kappaski

Architecture

Code architecture and runtime dataflow for the local-first Agent Runtime Control Plane.

Product shape: Kappaski is a control plane around existing agents. It does not replace the agent; it observes, decides, records, explains, verifies, and gates agent behavior.

End-To-End Control Loop

Preflight Scan→Session Registry→Invocation Normalize→Policy / Approval→Ledger→Proof→Gate

Native Integration Boundary

2026-05 correction: native plugins, hooks, extensions, rules, and MCP config are first-class adapter surfaces. They provide early intent and better UX, but the daemon remains the single-writer fact source and wrappers, proxies, shims, and sandbox layers remain the stronger enforcement path.

Runtime Components

CLI And Adapter Boundary

cli.py

Command surface for pre-runtime scanning, sessions, daemon authority, runtime events, proof export/verify, gate verify, and benchmark execution.

runtime.py

Normalizes loose runtime events into Invocation records, computes taint, calls policy/review, and appends action decisions to the ledger.

corpus.py

Scans pinned real Skill/MCP/tool snapshots, extracts capability/risk surfaces, and turns them into capability grant invocations.

Control Plane Core

daemon.py

File-backed local authority: session registry, lifecycle, single authority-mediated event writes, approval state, outcome state, heartbeat, and capability registration.

policy.py

Merges deterministic rule findings, semantic review, taint, policy mode, and profile into final effect and approval grade.

rules.py

Deterministic runtime and capability-surface rules. Critical deterministic findings cannot be downgraded by LLM review.

Evidence And Consumption

ledger.py

Append-only JSONL ledger with sequence numbers, previous hashes, and entry hashes.

postruntime.py

Exports portable proof reports from ledger-derived facts and verifies proof/ledger consistency.

gate.py

Consumes proof and ledger artifacts as release/CI policy gates: pass, warn, or fail.

evals.py

Regression suites for semantic decisions, real corpus capability grants, and proof gate consumption.

Engineering Module Layout

cli.py façade→parser family modules→commands/* handlers→domain modules→artifacts.py

The CLI compatibility surface is intentionally thin. commands/parser.py now composes parser family modules for foundation, governance, integrations, and product commands. Handler modules follow the same product families. evals.py remains the public benchmark façade while runner implementation is split under benchmarks/ by semantic baseline, foundation suites, release-version bands, full-product readiness, and registry mapping. Portable JSON/HTML outputs use artifacts.py; ledger JSONL remains owned by ledger.py.

Pre-v1 Control Plane Closure

Identity→Invocation→Path Graph→Path Policy→Mediation→Profile Governance→Pre-v1 Evaluation

v0.19-v0.24 add the missing pre-1.0 accountable path: principal/agent/grant facts, ledger-derived graph reconstruction, deterministic path policy, a unified mediation contract, governed policy profile objects, and a repeatable demo/benchmark package.

Release Candidate Closure

Adapter Runtime→Policy-as-Code→Evidence Bundle→Benchmark Registry→RC Gate

v0.25-v0.29 turn the pre-v1 model into a local release-candidate product surface: adapter executions produce complete packages, TOML policy profiles are validated and tested, enterprise evidence bundles are hash-verified, benchmarks are registered by product category, and the RC gate checks tests, docs, roadmap, benchmarks, demo artifacts, and evidence integrity.

Experiment Closure

Benchmark-Shaped Seed→Corpus Adapter→ExperimentCase→Simulated Agent Trace→Ledger / Proof / Evidence→Paper Suite

v0.30-v0.39 turn benchmark research into executable local experiment validation: benchmark-shaped LLM agent traces are normalized into ExperimentCase records, run through the same Kappaski control plane, and reported as reproducible E0-E6 artifacts. Official upstream/live benchmark validation is tracked separately.

Trust Boundary

Trusted Local Authority

Session registry, ledger append, policy evaluation, approval state, proof derivation, gate decisions.

Untrusted / Semi-Trusted Inputs

Agent output, Skill instructions, MCP/tool README files, shell commands, external content, user-provided runtime event payloads.
