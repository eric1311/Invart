v0.18 complete

Coverage-Aware Runtime

Proof, replay, and gate reports now describe Kappaski's actual control position: what was observed, mediated, enforced, or left as a coverage gap.

What Shipped

Coverage Core

src/kappaski/coverage.py defines grade order, dimensional records, merge helpers, and requirement comparison.

Proof Export

Runtime actions now carry coverage metadata. Proof exports aggregate coverage by dimension and event.

Gate

verify_gate can fail CI when a profile requires stronger coverage than the proof provides.

Replay

Replay timelines include observed/enforced coverage fields and a coverage summary block.

Coverage Grades

GradeMeaning
noneKappaski cannot see the behavior.
declaredThe capability is visible in preflight/config but not runtime.
observedThe event is visible after or alongside execution but not on the control path.
mediatedKappaski is on the execution path and can pause, approve, or deny.
enforcedA stronger wrapper/proxy/shim/sandbox boundary can block the real side effect.

kappaski gate verify --proof proof.json --ledger ledger.jsonl --mode ci --session-profile profile.json
kappaski replay export --ledger ledger.jsonl --out replay.html

Boundary

Coverage-aware reporting does not pretend advisory observation is enforcement. Enterprise signoff remains intentionally deferred beyond v0.18; this version builds the factual substrate signoff would later consume.
