Kappaski

v0.5 Proof Gate

Proof consumption and CI/review gate design: turning runtime evidence into downstream governance decisions.

v0.5 goal: make Kappaski proof actionable outside the session. A proof that cannot fail a release, CI check, or review is only a report.

Gate Pipeline

ledger.jsonl+proof.json→Verify Integrity→Apply Gate Mode→GateReport→CI / Review

Gate Modes

ModeUse CaseBehavior
auditLocal analysis and early rolloutIntegrity failures fail. Missing approvals and unresolved grants warn.
managedDefault local/team governanceMissing approvals, rejected approvals, and high-risk unresolved grants fail.
ciPR or release gateRequires both proof and ledger. Proof mismatch, invalid ledger, missing approvals, high-risk unresolved grants, and risky tainted allows fail.

Implemented CLI

kappaski gate verify --proof proof.json --ledger ledger.jsonl --mode managed
kappaski gate verify --proof proof.json --ledger ledger.jsonl --mode ci --out gate-report.json
kappaski eval benchmark --suite v0.5-proof-gate

Gate Checks

Integrity

Proof schema, ledger hash chain, proof/ledger derived summary match.

Approval Closure

Required approvals must be approved; missing or rejected approvals fail in managed and CI modes.

Capability Closure

High-risk Skill/MCP/tool capability grants must not remain unresolved.

Taint Closure

CI mode fails if a tainted session contains allowed medium-or-higher risky actions.
