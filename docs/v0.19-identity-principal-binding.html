v0.19 Identity & Principal Binding

Accountability facts for who authorized which agent, grant, and credential boundary.

Product Goal

Every governed session can answer: who is accountable, which agent acted, which credential boundary existed, and which capability grant authorized an invocation.

Data Flow

Principal->AgentIdentity->CredentialBinding->CapabilityGrant->Ledger->Proof Accountability

CLI

kappaski identity declare --principal alice@example.com --out identity.json
kappaski identity bind --ledger ledger.jsonl --session ks_123 --principal alice@example.com --agent claude-code --scope file_read --resource /repo/.env --env-key OPENAI_API_KEY
kappaski identity inspect --ledger ledger.jsonl

Validation

CheckExpected
Managed daemon missing principalRejected before session registry write
Declared agent mismatchRejected with identity mismatch
Proof exportIncludes principal, agent identity, credential boundary, and grants
