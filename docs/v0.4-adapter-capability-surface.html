Kappaski

v0.4 Adapter And Capability Surface

Real Corpus First design for Skill/tool/MCP capability scanning, authority-mediated capability grants, policy approval, and proof export.

Hard constraint: v0.4 scanner validation must use pinned snapshots of real public Skill/tool/MCP corpora. Mock-only validation is not acceptable.

Architecture

Pinned Corpus→Static Scanner→Capability Surface→Capability Grant→Policy + Approval→Ledger + Proof

Corpus Sources

  • Public SKILL.md repositories
  • Claude-compatible skills
  • MCP server repositories
  • Tool manifests and README files

Capabilities

  • shell, file read/write, network
  • browser, git, cloud
  • messaging, payment, database, calendar
  • MCP/tool server surface

Risk Signals

  • credential references
  • external writes
  • destructive actions
  • shell execution
  • external dependency and target deviation

Implemented CLI

kappaski corpus scan --root benchmarks/corpora
kappaski daemon register-capabilities --target . --session <id> --corpus-root benchmarks/corpora --adapter codex-wrapper
kappaski eval benchmark --suite v0.4-real-skill-surface

Closed Loop Behavior

Stagev0.4 behavior
ScanRead pinned public snapshots only; compute content hashes and detect capability/risk surface.
GrantCreate deterministic capability grant ids from adapter, source id, kind, and content hash.
DecideRecord each grant as a session invocation; high-risk surfaces require approval under managed policy.
PersistWrite grants, findings, policy decisions, and pending approvals to the authority-owned ledger.
ProveExport capability_grants in proof summary and verify the ledger-derived counts.
