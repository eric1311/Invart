Kappaski

v0.6 Real Workflow Demo

Reference adapter wrapper and CI artifact flow for a reproducible Kappaski control-plane demo.

v0.6 goal: connect the local control plane to an actual wrapper workflow that emits ledger, proof, and gate report artifacts.

Wrapper Flow

adapter run→Create Authority Session→Register Capability Corpus→Run Child Command→Close Session→Export Proof→Gate

Implemented CLI

kappaski adapter run \
  --target . \
  --agent codex \
  --goal "Run under Kappaski" \
  --out-dir .kappaski/demo \
  --capabilities audit \
  --gate ci \
  -- python3 -c "print('ok')"

kappaski eval benchmark --suite v0.6-adapter-workflow

Artifacts

ArtifactProducerConsumerRole
ledger.jsonlRuntime authorityProof exporter, gateAppend-only source of truth.
proof.jsonPost-runtime exporterGate, reviewers, CIPortable summary derived from the ledger.
gate-report.jsonProof gateCI, PR checks, enterprise control planeMachine-readable pass/warn/fail decision.

Capability Modes

ModeBehavior
offDo not register the pinned corpus before the child command.
auditRegister capability surfaces as audit facts. Useful for clean demo and CI pass path.
managedHigh-risk capability surfaces require approval. Useful for demonstrating gate failure.

CI Template

The repository includes .github/workflows/kappaski-proof-gate.yml. It installs Kappaski, verifies proof plus ledger, runs gate verify --mode ci, and uploads proof artifacts.
