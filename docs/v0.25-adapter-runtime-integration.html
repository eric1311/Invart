v0.25 Adapter Runtime Integration

Adapter execution is now a full control-plane run, not just a command wrapper.

Goal

Run Claude Code style and generic adapters through session registry, local identity binding, unified mediation, and post-runtime evidence export.

Data Flow

adapter runtime->session registry->identity binding->mediation->child command->proof/replay/graph/coverage/audit/package

Artifacts

ledger.jsonl
proof.json
replay.html
path-graph.html
coverage.html
audit.html
adapter-package.json

CLI

kappaski adapter run --target . --out-dir .kappaski/run --gate managed -- python3 -m pytest -q
kappaski adapter inspect --package .kappaski/run/adapter-package.json
kappaski adapter package --ledger .kappaski/run/ledger.jsonl --out-dir .kappaski/evidence

Tests

PYTHONPATH=src /usr/bin/python3 -m pytest -q tests/test_core.py::test_v025_adapter_runtime_package_closes_product_loop
PYTHONPATH=src python3 -m kappaski.cli eval benchmark --suite v0.25-adapter-runtime-integration

Boundary

This is local wrapper/runtime integration. It does not claim kernel-level capture, hosted adapter fleet management, or IdP-backed agent identity.
