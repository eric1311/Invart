v0.9 Harness Compatibility

SWE-Bench Lite compatibility target and baseline-vs-wrapped harness report model.

Implemented Scope

  • Compatibility means same exit code, same expected artifacts, and same grading result.
  • Minor metadata differences are allowed and reported.
  • Managed mode may pause a harness run, but continuation requires human approval.
  • Default tests stay lightweight; full SWE-Bench harness execution remains optional heavy validation, but the CLI now has a dedicated official harness entrypoint.
  • The runner can compare supplied baseline/wrapped artifacts, execute real baseline/wrapped command pairs, or invoke the official SWE-Bench Lite harness when Docker and swebench are installed.

Verification

kappaski harness compare --baseline baseline.json --wrapped wrapped.json --case benchmarks/cases/swe-bench-lite/pinned_cases.json
kappaski harness swe-bench-lite --case benchmarks/cases/swe-bench-lite/pinned_cases.json --baseline-artifact baseline.json --wrapped-artifact wrapped.json --out report.json
kappaski harness swe-bench-lite --case benchmarks/cases/swe-bench-lite/pinned_cases.json --skip-if-unavailable --out skipped.json
kappaski harness swe-bench-official --python /tmp/kappaski-swebench-py311-venv/bin/python --instance-id django__django-11001 --predictions-path gold --run-id kappaski_smoke --report-dir /tmp/kappaski-swebench-report --timeout 60 --out official.json
kappaski eval benchmark --suite v0.9-harness-compatibility

Boundary

This version remains a CLI/control-plane slice, but it has now been smoke validated against the official SWE-Bench Lite harness on django__django-11001 with gold predictions: one completed instance, one resolved instance, and zero errors. Managed pause/approval/resume is implemented in the local managed harness check; full official Docker/dataset runs remain optional heavy validation.
