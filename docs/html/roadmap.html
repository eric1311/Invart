Invart Roadmap

Open-source path to 1.0 readiness

This roadmap covers public product work: daemon, adapters, registration, policy, evidence, validation, and benchmark experiments. It excludes local-only planning artifacts.

Roadmap Principles

PrincipleMeaning for development
Ledger firstRuntime facts enter an append-only ledger. Proof, replay, graph, audit, and evidence are derived artifacts.
Coverage honestyObserved, mediated, enforced, fail-open, degraded, imported, and discovered positions remain distinct.
Real agent evidenceAdapter claims need live or fixture-backed runtime artifacts, not only product documentation.
Low-friction governanceBenign coding workflows should continue with low approval noise and measured overhead.
Test-driven deliveryEach stage starts with product-level tests and ends with benchmark, docs, and release-gate validation.
No false enterprise claimEnterprise control-plane language requires registration, enrollment, policy, and unmanaged-gap evidence.

Current Baseline

Available Now

  • CLI-first managed sessions, ledger, proof, replay, graph, coverage, audit, evidence bundles, and RC checks.
  • Generic adapter wrapper and Claude Code reference adapter.
  • Adapter profiles with truthful coverage grades.
  • L1-L5 runtime workflow and L5 evidence workspace.
  • Local benchmark suites and progressive external evidence attachment.

Not Yet Claimed

  • Hosted enterprise admin console.
  • IdP or SCIM-backed organization service.
  • OS or kernel-level sandbox.
  • Guaranteed control over unmanaged agents that bypass Invart.
  • Full live adapter for every agent product.

Coverage Grade Vocabulary

GradeMeaningCannot claim
full_live_adapterReal installed agent enters Invart through managed launch, daemon session registration, mediation, and L5 artifact export.Universal OS-level containment or complete vendor-private visibility.
managed_wrapperInvart controls launch and visible command/file/network-shaped workflow boundaries.Fine-grained native hook coverage unless real native events enter Invart.
native_bridgeVendor hook, plugin, extension, MCP, or SDK events enter Invart and can receive normalized decisions.Full runtime enforcement beyond the covered native event surface.
vendor_evidence_importInvart imports vendor logs, sandbox facts, approval records, traces, PRs, or backend reports.Invart mediation or enforcement of the original side effect.
discovery_inventoryInvart discovers binaries, configs, plugins, skills, MCP servers, or unmanaged surfaces.Runtime control.

Detailed Stage Work

v0.9.8 Claude Code Full Live Adapter

Development: real binary detection, daemon-backed session creation, principal and agent identity binding, managed launch, hook/tool event normalization, mediation, approval or deny, outcome capture, and full L5 export.

Tests: strict live failure on missing binary, fixture mode distinction, benign autonomy, risk intervention, degraded coverage reporting.

Validation: adapter package inspection, evidence workspace verification, live run artifact completeness.

Experiments: baseline versus managed benign run, dummy secret egress or unsafe deletion, hook-mediated versus wrapper-mediated comparison.

v0.9.9 Conformance Contract v2

Development: evidence level, control position, side-effect timing, required artifacts, source citation, and gate rules for every adapter profile.

Tests: grade-specific artifacts, imported evidence cannot satisfy mediated/enforced claims, coverage label failure on missing evidence.

Validation: conformance reports feed CLI, benchmark, product matrix, and release gate.

Experiments: same-action raw, trace-only, wrapper, bridge, shim, and vendor-import comparison.

v0.9.10 OpenCode Real Adapter Track

Development: OpenCode binary detection, plugin/config inventory, managed wrapper, mediation for visible actions, full adapter artifacts where possible.

Tests: truthful profile grade, fixture run, pre-runtime plugin inventory, managed risk action.

Validation: live run when installed, blocked/pending when missing, product matrix artifact links.

Experiments: benign repo task, plugin or skill scan, risk-equivalent command/file action.

v0.9.11 Terminal Agent Managed Wrapper Track

Development: Gemini CLI and Aider profiles, binary probes, config/MCP/context inventory, direct versus managed artifact comparison.

Tests: managed-wrapper or lower grade unless hook evidence exists, exit-code and artifact parity, approval noise measurement.

Validation: compatibility report for installed or fixture-backed agents.

Experiments: Gemini MCP/config run, Aider edit/test workflow, paired benign friction runs.

v0.9.12 Codex Deep Adapter

Development: Codex detection, managed wrapper, vendor-native sandbox/approval/network evidence import, non-inflation gate checks.

Tests: Codex-native controls stay vendor evidence, managed wrapper actions produce Invart decisions, gate catches upgraded claims.

Validation: direct versus managed comparison and honest strict-mode failures.

Experiments: benign compatibility, risk-equivalent managed run, vendor-native evidence import row.

v0.9.13 IDE Bridge And Inventory

Development: Cline, Roo, and Cursor profiles for extension, rule, MCP, and settings surfaces with optional bridged/imported events.

Tests: discovery-only cannot satisfy mediation, imported events preserve source and limitation, blind spots remain visible.

Validation: IDE inventory report and evidence bundle import.

Experiments: config/MCP/rules inventory, event sample import, discovery-only negative gate.

v0.9.14 Gateway And Server-Agent Evidence

Development: OpenClaw and Hermes profiles for gateway, security, skill, container, backend, and command evidence fields.

Tests: vendor-owned evidence stays audit/import coverage, source hash and limitation are preserved, managed mode emits normal artifacts when available.

Validation: report shows managed, imported, or discovery-only grade per row.

Experiments: security posture inventory, safe-equivalent risk case, vendor evidence import case.

v0.9.15 Enterprise Registration Authority

Development: agent catalog, launcher registry, adapter registry, registration policy, declared identity checks, unmanaged discovery reports.

Tests: registered launch accepted, identity mismatch rejected, unregistered agent reported as gap, full-control claim requires launcher or adapter evidence.

Validation: daemon status and release gate can require registration evidence.

Experiments: registered versus unregistered comparison, mismatched identity, mixed workspace coverage-gap scan.

v0.9.16 Public Benchmark Slice

Development: small reproducible public benchmark slice mapped to control-plane metrics.

Tests: every case emits full L5 artifacts, pinned fixtures are stable, external dependencies skip cleanly unless strict.

Validation: benchmark reports block rate, approval rate, false-positive proxy, proof completeness, coverage distribution, and reconstruction status.

Experiments: indirect instruction, risky tool/code action, skill/plugin poisoning, benign control.

v0.9.17 Benign Coding Friction

Development: paired baseline versus managed workflow runner across priority agents where installed or fixture-backed.

Tests: separate baseline and managed artifacts, approval noise from ledger decisions, compatibility fails on unexplained artifact divergence.

Validation: compatibility report with exit code, artifact, grading result, metadata differences, approval count, overhead, and evidence completeness.

Experiments: repo inspection, test command, small edit/test workflow, SWE-Bench-Lite-shaped sample when available.

Cross-Cutting Test Strategy

Test familyProduct question
Profile contract testsWhat does Invart claim for this product, and what evidence is required?
Adapter workflow testsCan a real or fixture agent enter Invart and produce ledger/proof/evidence?
Risk intervention testsDoes Invart block or pause risky side effects before they happen on covered surfaces?
Benign autonomy testsDoes Invart avoid excessive approval noise for ordinary work?
Coverage truthfulness testsAre observed, mediated, enforced, imported, and discovered positions kept distinct?
Evidence integrity testsCan a reviewer trust the bundle and detect missing or tampered artifacts?
Release gate testsDo missing docs, tests, benchmarks, registration, or evidence fail the right gate?

Validation Commands

PYTHONPATH=src uv run --with pytest pytest -q
PYTHONPATH=src python -m invart.cli roadmap status --require-full
PYTHONPATH=src python -m invart.cli eval benchmark --suite full-product-readiness
PYTHONPATH=src python -m invart.cli release-candidate verify --out-dir .invart/rc --skip-pytest

Strict live validation must fail when required live evidence is absent. Missing local binaries are never reported as a successful live run.

PYTHONPATH=src python -m invart.cli real-agent check --require-live --out-dir .invart/live-agent-check
PYTHONPATH=src python -m invart.cli real-agent run --agent claude-code --require-live --out-dir .invart/live-claude

Deferred Beyond This Roadmap

Hosted enterprise admin console, IdP/SCIM/SAML integration, production SIEM or OTel export, kernel-level enforcement, graph database backend, full signoff workflow, and local-only planning artifacts remain outside this open-source roadmap slice.
