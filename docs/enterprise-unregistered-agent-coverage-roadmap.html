Unregistered Agent Discovery And Coverage Roadmap

企业环境里不能假设所有 agent 都会主动通过 Kappaski daemon 注册。这个文档定义“已注册、已发现、已调解、已强制”的覆盖模型，以及后续 roadmap 如何尽量收口未注册 agent。

核心判断

Kappaski daemon 的 session registry 是最干净的事实入口，但真实企业环境一定会出现 bypass：用户直接启动 CLI agent、IDE extension 自带 agent、CI runner 中的 agent、临时脚本、MCP server 或浏览器插件。Kappaski 不能在没有 OS/EDR/MDM 权限时声称完全覆盖，但可以做三件事：发现、标记 coverage gap、把高风险入口迁移到可调解路径。

Registered→Discovered→Mediated→Enforced→Audited

覆盖等级

等级含义企业风险Kappaski 行为
unseen没有任何信号。完全盲区。只能通过 endpoint/network/MDM/EDR 集成降低概率。
unmanaged_detected发现疑似 agent 进程、配置或网络行为，但未注册。可审计性弱，不能证明路径完整。生成 coverage finding，建议迁移到 managed launcher。
observed能看到部分日志、hook、network 或 file signals。可回顾但不一定可阻断。进入 ledger，但 proof 不得声称 mediated/enforced。
mediated通过 wrapper/proxy/hook 进入统一 mediation contract。能 pause/approval/deny，但底层仍可能 fail-open。写入 decision/outcome/approval facts。
enforced关键动作被 shim/proxy/OS/endpoint policy 强制。高价值企业目标。proof/gate 可以声明 enforcement 覆盖。

未注册 Agent 的发现信号

信号可发现什么限制
Process inventoryclaude、codex、gemini、opencode、IDE child process、node/python agent launcher。只能看到进程名/命令行，不一定能看到内部 tool calls。
Agent config scan.claude、.codex、MCP config、IDE extension settings、skill directories。静态配置不等于真实运行。
Shell/launcher integrationshell alias、PATH shadowing、managed launcher、CI wrapper。用户可以绕过；企业需要 MDM/endpoint policy 辅助。
Network/proxy/DNS logs模型 API、MCP server、GitHub/GitLab upload、unknown exfil endpoint。只能证明网络行为，不能完整解释 agent intention。
File activitysecret file read、workspace deletion、bulk rewrite、credential store access。无 endpoint 权限时只能在 wrapper/shim 范围内观察。
CI runner hooksworkflow 中启动的 agent、deploy mutation、artifact upload。需要 CI 管理权限。

Roadmap 提案

版本目标测试设计
v0.41Unmanaged agent inventory：扫描进程快照、repo/home agent config、MCP config、skill/plugin dirs，输出 unmanaged coverage report。用 fixture 生成 Claude/Codex/Gemini/OpenCode/OpenClaw 风格配置，验证发现率和 false-positive proxy。
v0.42Managed launcher migration：为已发现 agent 生成 wrapper/alias/CI launcher 建议，并能一键 dry-run 安装。验证安装 preview 不破坏原命令，wrapped run 进入 mediation contract。
v0.43Enterprise coverage gate：根据 profile 要求阻断或告警未注册 agent、weak coverage 和 bypass path。构造 registered / unmanaged_detected / mediated / enforced 四类 session，验证 gate 结果和 audit 声明一致。
post-v0.43Endpoint/MDM/EDR/SIEM connectors：把发现信号接入企业资产系统。只在真实企业/实验环境中算 external validation，不进入默认本地 pass。

开发原则

  • 不要把“发现了疑似 agent”说成“控制了 agent”。
  • 不要把网络/进程外部观察说成完整 trajectory。
  • 所有未注册 agent finding 必须能进入 proof/audit/gate，并明确 coverage grade。
  • 企业模式可以要求 managed launcher；个人模式只建议迁移，不默认阻断。
