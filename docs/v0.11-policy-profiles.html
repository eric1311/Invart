v0.11 Policy Profiles

TOML-first policy profile resolution with session > repo > team precedence.

Implemented Scope

  • TOML is the default human-editable profile format; JSON is accepted for machine/API paths.
  • Precedence is session > repo > team.
  • Enterprise mode disallows local override by default.
  • Break-glass override is now ledger-backed with actor, reason, scope, expiry, and immutable profile_override entry.
  • Profile files are threaded through runtime, gate, replay, and enforcement CLI paths.

Verification

kappaski profile resolve --team team.toml --repo repo.toml --session session.toml
kappaski profile break-glass --ledger ledger.jsonl --session ks_demo --actor security-admin --reason "incident" --scope repo
kappaski gate verify --ledger ledger.jsonl --session-profile session.toml
kappaski eval benchmark --suite v0.11-policy-profiles

Boundary

This version is an implemented local control-plane product surface. It records the product contract and validates behavior through deterministic tests and built-in benchmarks before deeper native or external harness integration.
