Kappaski

Product Decisions

Accepted product decisions and version-boundary principles for the Agent Runtime Control Plane.

Positioning: Kappaski sits outside agent apps such as Codex, Claude Code, Cursor, OpenClaw, and future domain agents. It provides unified observation, governance, audit, and risk analysis.

Product Identity

Agent Runtime Control Plane

Coding agents are the first wedge, not the final boundary.

Moat

Turn agent behavior into governed runtime transactions across the full lifecycle.

Proof / Ledger

Ledger is the fact source. Proof is the portable summary. Gate consumes proof plus ledger.

Policy Modes

ModeBehavior
auditRecord and evaluate without blocking. This is the most open mode.
advisoryProduce decisions and record overrides.
managedPause or block high-risk actions until resolution.
ciUse proof verification as a policy gate, but do not assume the session must be closed.

Open Approval Mode

Bulk approval is allowed in v0.7 because it represents the intentionally open operating mode. It must still write one approval evidence record per decision and require a reason. Later enterprise policy can restrict or disable bulk approval by mode.

Partial Visibility Decision

Gate must not require a closed session by default. Enterprise environments should avoid partial blind periods. A gate may evaluate staged proof while a run is still active so operators can catch risk early. --require-closed-session may exist as an optional strict control, but it should not be assumed for CI.

Replay Disclosure

Replay reports should include command, path, and URL details. Raw content is not discarded by default, but it should be folded/collapsed and prepared for stricter redaction modes later.

Plugin And Extension Boundary

Accepted 2026-05 correction: Kappaski is plugin-assisted and hook-aware, not plugin-only. Native plugins, hooks, extensions, rules, and MCP config are first-class integration surfaces for UX, early intent, tool metadata, and pre-runtime discovery. They still do not own policy, storage, replay, or the final enforcement claim; the daemon remains the product core and single-writer fact source.

Promoted

Claude Code hooks, Codex hooks/plugins, OpenCode plugins, Gemini extensions, Cursor rules/hooks/MCP, and MCP broker support.

Still Required

Wrapper/proxy/shim/sandbox layers for durable evidence, subprocess effects, file/network enforcement, and cross-agent consistency.

Reference: Plugin And Extension Architecture Review.

MCP Proxy Priority

MCP proxy priority is revised from low to medium-high. MCP is not the product core, but it is now an important cross-agent control path for tool-call audit, approval, redaction, and compatibility across Claude Code, Codex, Gemini CLI, Cursor, OpenCode, and related agents.

Coverage Grade

Coverage grade is not a risk score. It describes Kappaski's actual control position for an action class or runtime event: none, declared, observed, mediated, or enforced. Coverage is recorded by dimension: preflight visibility, runtime observation, runtime enforcement, and post-runtime audit.
ModeCoverage Gap Behavior
auditWarn and record the gap.
managedRequire human approval or explicit policy resolution when the profile requires stronger coverage.
ciFail when the profile requires stronger coverage.

Enterprise signoff is intentionally deferred beyond v0.18; v0.18 focuses on coverage-aware proof, replay, audit, and gate behavior.

LLM Reviewer Boundary

Can Do

  • Auto-grade low risk actions
  • Upgrade semantic risk
  • Explain ambiguity

Cannot Do

  • Downgrade deterministic critical rules
  • Become root of trust
  • Receive raw secrets by default

v0.8-v0.14 Accepted Implementation Line

v0.8 through v0.14 are now local CLI/control-plane slices: OpenAI-compatible LLM reviewer, SWE-Bench Lite harness compatibility reporting, Claude Code adapter profile, TOML/JSON policy profiles, multi-user TeamRun/Handoff, enforcement guard decisions plus a source-level Rust file-write shim contract, and an enterprise audit demo package. Rust shim compilation, wrapper-level file-write interception, official SWE-Bench smoke execution, and real Claude binary checks are now validated in the local environment. Kernel/OS-level blocking, live real-agent demo execution, managed harness pause/resume, and enterprise signoff remain later hardening layers.
