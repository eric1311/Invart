Kappaski Code Design And Development Standards

这份文档记录当前代码重构判断、模块边界、测试驱动规范和后续开发约束。目标是让 Kappaski 从“版本功能堆叠”稳定收口为可维护、可解释、可验证的 control-plane 产品。

当前重构结论

当前代码已经具备完整本地产品闭环，但增长压力集中在三个地方：src/kappaski/cli.py 聚合了过多命令解析和 handler，src/kappaski/evals.py 聚合了过多 benchmark dispatch，tests/test_core.py 聚合了过多版本测试。短期不能为了“漂亮切片”大规模搬动已有稳定功能，应该先做低风险的结构化收口。

问题本次处理后续规范
benchmark dispatch 是长 if-chain改为 registry-style runner dispatch，新增 suite 只注册 runner。新增 benchmark 必须有 runner、registry entry、CLI test、docs。
真实 demo 数据散落在文档叙述中新增 real_world_cases.py，把公开来源、claim boundary、before/during/after 映射结构化。所有真实世界 claim 必须进入可导出的 JSON，而不是只写在 HTML 里。
文档和代码容易脱节新 demo 同时提供 CLI、benchmark、HTML 入口和测试。版本完成必须同步更新 docs index、user guide/briefing 或 roadmap。

推荐模块边界

CLI parser→Command handler→Domain service→Ledger / Artifact writer→Benchmark / Gate
层职责不要做什么
CLI parser只负责参数、帮助文本、基本枚举。不要直接拼装复杂业务对象或写 artifact。
Command handler把参数转成 domain service 调用，并打印 JSON。不要重复策略判断、风险判断、hash 逻辑。
Domain service实现 identity、policy、mediation、graph、evidence、demo 等产品能力。不要依赖 stdout 或 argparse。
Artifact writer稳定写 JSON/HTML/JSONL，保证路径、hash、schema 可复现。不要把 claim boundary 只藏在页面文案里。
Benchmark/Gate验证产品问题是否被回答，而不是只测函数返回。不要把 optional heavy skip 伪装成 external validation pass。

代码规范

Schema First

新增可导出对象必须有 schema_version，稳定 JSON 输出要使用 sort_keys=True。

Ledger First

ledger 是事实源；proof、replay、graph、audit、bundle 都是派生表示。

Coverage Honest

observed、mediated、enforced、fail_open_alert 不得混淆。

Claim Boundary

公开报告映射、模拟 fixture、contract test、official benchmark run 必须明确区分。

LLM Boundary

LLM reviewer 可升级风险和生成说明，但不能降级 deterministic critical rule。

HTML Docs

面向用户、架构、roadmap 和版本完成说明优先写 HTML，并补充图形化结构。

TDD 完成定义

每个新能力必须先回答产品问题，再谈实现细节。

1. 写失败测试：CLI、domain service、artifact、benchmark 至少覆盖两个层面。
2. 实现最小能力：不绕过 ledger、proof、coverage、claim boundary。
3. 补 benchmark：必须输出 metrics 或 checks，能被 eval benchmark 调用。
4. 补文档：HTML version/architecture/user-facing entry 至少更新一处。
5. 跑验证：pytest、对应 benchmark、roadmap/RC gate 或 targeted gate。
6. 不允许把 mock 结果当作真实外部验证；optional heavy 必须 clean skip。

重构路线

优先级任务验收
P0新增真实世界风险案例模块和 demo benchmark。eval benchmark --suite real-world-agent-risk-demo 通过。
P0benchmark dispatch 注册表化。新增 suite 不再追加长 if-chain。
P1把 cli.py 拆成 commands/ 包，保持 kappaski.cli.main 兼容。所有 CLI 测试不变；每个 command family 单独测试。
P1把 tests/test_core.py 拆成 product slice：runtime、policy、adapter、evidence、benchmark、docs。测试名和覆盖意图更容易定位。
P2统一 HTML/JSON artifact writer helper。demo、audit、evidence、report path 行为一致。
